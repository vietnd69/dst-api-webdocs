# Veggies

> Based on game build **714014** | Last updated: 2026-03-07

## Overview
This file is not a component itself but a **prefab factory** that defines `VEGGIES` data and the `MakeVeggie` function to dynamically generate prefabs for all standard vegetables in DST. It uses Tuning values and external definitions from `farm_plant_defs.lua` to configure multiple variants per vegetable: pristine (fresh), cooked, dried (where applicable), oversized (for the Mighty Gym), oversized waxed, and oversized rotten. The prefabs are equipped with appropriate components (`edible`, `perishable`, `cookable`, `deployable`, `equippable`, `weighable`, `lootdropper`, etc.) depending on variant and vegetable type.

## Usage example
```lua
-- Load and register all vegetable prefabs
local prefabs = require("prefabs/veggies")

-- prefabs is a flat array of Prefab objects
-- EachPrefabPreInit(prefab) in main.lua will register them
```

## Main functions
### `MakeVegStats(...)`
* **Description:** Returns a table of stats (health, hunger, sanity, perish times, float settings, etc.) for a vegetable, used to configure `VEGGIES` entries.
* **Parameters:**
  * `seedweight` (number) — Seed drop probability or rarity.
  * `hunger`, `health`, `perishtime`, `sanity` — Raw food values.
  * `cooked_hunger`, `cooked_health`, `cooked_perishtime`, `cooked_sanity` — Cooked food values.
  * `float_settings` — Raw float config: {`size`, `float_chance`, `sink_chance`}.
  * `cooked_float_settings` — Cooked float config.
  * `dryable` — Drying config table (or `nil`).
  * `secondary_foodtype` — `FOODTYPE` enum or `nil`.
  * `halloweenmoonmutable_settings` — Mutation config or `nil`.
  * `lure_data` — Fishing lure data or `nil`.
  * `issnowmandecor` — Boolean for snowman decoration eligibility.
* **Returns:** Table with all numeric and config fields as documented above.

### `MakeVeggie(name, has_seeds)`
* **Description:** Factory function that creates and returns an array of `Prefab` objects for a given vegetable name (e.g., `"carrot"`), including fresh, cooked, dried, oversized, oversized waxed, oversized rotten, and seeds (if `has_seeds` is `true`). Sets up assets, components, and post-initialization hooks.
* **Parameters:**
  * `name` (string) — Internal prefab name (key in `VEGGIES`).
  * `has_seeds` (boolean) — Whether to generate a seeds prefab and attach seed-compatible components.
* **Returns:** Array of `Prefab` objects (one per variant: `[name.."_seeds", name, name.."_cooked", name.."_dried"?, name.."_oversized", name.."_oversized_waxed", name.."_oversized_rotten"]`).
* **Error states:** Does not fail; returns empty array if no variants are defined for the vegetable.

### `can_plant_seed(inst, pt, mouseover, deployer)`
* **Description:** Validation function used by `deployable` component to determine if a seed can be planted at a given point.
* **Parameters:**
  * `inst` — Seed instance.
  * `pt` — Position table `{x, y, z}`.
  * `mouseover`, `deployer` — Unused in current implementation.
* **Returns:** Boolean — `true` if soil at `pt` can be tilled (`TheWorld.Map:CanTillSoilAtPoint(x,0,z,true)`), otherwise `false`.

### `OnDeploy(inst, pt, deployer)`
* **Description:** Deploys a seed to spawn the corresponding plant prefab (e.g., `"farm_plant_carrot"`), sets its position, pushes `on_planted` event, collapses the soil, and removes the seed.
* **Parameters:**
  * `inst` — Seed instance.
  * `pt` — Target position table `{x, y, z}`.
  * `deployer` — Entity placing the seed.
* **Returns:** Nothing.

### `oversized_makeloots(inst, name)`
* **Description:** Generates default loot table for oversized vegetables before work/completion.通常 includes two copies of the veggie name, two seeds, and a mixed fifth item.
* **Parameters:**
  * `inst` — Oversized vegetable instance.
  * `name` (string) — Vegetable name.
* **Returns:** Array of prefabs, e.g., `{name, name, name.."_seeds", name.."_seeds", name_or_seeds}`.

### `oversized_onperish(inst)`
* **Description:** Handles spoilage of oversized vegetables. If placed on a non-Gym, drops `spoiled_food`. If on a Gym, spawns `"veggie_overridden"_rotten`, maintains inventory slot, and reloads weight into the gym.
* **Parameters:**
  * `inst` — Oversized vegetable instance.
* **Returns:** Nothing.

### `dowaxfn(inst, doer, waxitem)`
* **Description:** Converts a waxable oversized veggie into its `_waxed` variant, preserving equipped state (e.g., body slot).
* **Parameters:**
  * `inst` — Raw oversized veggie instance.
  * `doer` — Player performing waxing.
  * `waxitem` — Wax item used (unused in logic).
* **Returns:** `true`.

### `Carrot_StartSpinning(inst, time)`
* **Description:** Initiates a spinning animation/sound effect for the carrot (event-specific behavior).
* **Parameters:**
  * `inst` — Carrot instance.
  * `time` (number?) — Spin duration in seconds (default `2`).
* **Returns:** Nothing.

### `Carrot_StopSpinning(inst)`
* **Description:** Stops carrot spinning animation, sound, and timer, resets facing and rotation.
* **Parameters:**
  * `inst` — Carrot instance.
* **Returns:** Nothing.

### `Carrot_TimerDone(inst, data)`
* **Description:** Called when the carrot spin timer completes; spawns a `carrot_spinner` FX and resets activatable state.
* **Parameters:**
  * `inst` — Carrot instance.
  * `data` — Timer event data (must be `{name="spin"}`).
* **Returns:** Nothing.

### `Seed_GetDisplayName(inst)`
* **Description:** Returns a localized name for seeds if the plant is known in the plant registry.
* **Parameters:**
  * `inst` — Seed instance.
* **Returns:** Localized string or `nil`.

### `Oversized_OnSave(inst, data)`
* **Description:** Saves oversized veggie state (harvest day, plant origin).
* **Parameters:**
  * `inst` — Oversized veggie instance.
  * `data` — Table to write to.
* **Returns:** Nothing.

### `Oversized_OnPreLoad(inst, data)`
* **Description:** Restores oversized veggie state from `data` during load.
* **Parameters:**
  * `inst` — Oversized veggie instance.
  * `data` — Saved table.
* **Returns:** Nothing.
